Development Workflow — State Machine

Azure DevOps Board states · Agent orchestration via Hermes Kanban · v2

Your Actions Per Ticket

  • • Move to Ready for Design (new ticket + PRD)
  • • Move to Ready for Development (plan approved)
  • • At Reviewed: move to Done (approved)
  • • At Reviewed: move to Ready for Dev (fixes needed)
  • • Merge PR when satisfied

Agent Profiles

  • • default — ADO poller + Discord + cleanup
  • • planner — PRD → plan.md (OpenCode plan)
  • • coder — plan → code → PR (OpenCode build)
  • • reviewer — plan vs diff → threads (OpenCode plan, variant max)

PR Thread Lifecycle

  • • Reviewer posts inline threads (active)
  • • You move back to Ready for Development
  • • Coder reads threads, addresses each in code
  • • Coder replies to thread + marks Fixed
  • • Reviewer re-reviews, re-votes, back to Reviewed
  • • You control every cycle — no automatic loops
